The NS1 protein is a
key target for dengue antivirals.
Exploring 33,000 compounds by hand

Dengue fever affects over 100 million people annually. The NS1 protein, central to viral replication, has emerged as a high-value target for antiviral drug design — but the chemical space of potential inhibitors is enormous.

Traditional screening — testing each compound individually — is slow, expensive, and often misses structurally relevant candidates. We needed a smarter filter.

02 / 06 Why Graphs?

A molecule is a graph.
Why not treat it like one?

There are existing methods for comparing molecules — called fingerprints — that encode chemical structure as binary vectors. They work by asking yes/no questions: does this fragment exist? Is that ring present?

Fingerprints are fast. But they throw away exactly what matters for binding: the arrangement in 3D space, the connectivity of functional groups, and the spatial relationships between pharmacophoric features.

Graphs, by contrast, encode those relationships natively. Atoms become nodes. Bonds become edges. Two molecules with the same atoms but different arrangements produce structurally different graphs — and therefore different similarity scores.

03 / 06 Extended Reduced Graphs

Atom-level graphs are
too noisy. We need an abstraction
that speaks pharmacology.

Working with full atomic graphs — hundreds of nodes per molecule — introduces irrelevant structural noise. Extended Reduced Graphs (ErG) solve this by replacing groups of atoms with single pharmacophoric nodes that represent their biological role at the NS1 binding site.

04 / 06 Measuring Structural Similarity

How much does it cost to transform
one molecule's graph into another?

Graph Edit Distance (GED) answers this precisely. It counts the minimum number of operations — node substitutions, insertions, deletions, edge changes — required to transform one ErG into another. Lower GED = higher structural similarity to our NS1 reference inhibitor.

Internally, GED uses the Hungarian algorithm to find the optimal matching between nodes of two graphs — then builds the full edit cost on top of that assignment.

The Hungarian algorithm guarantees the optimal node-to-node matching in O(n³) time — but alone it only solves the assignment problem. GED uses that matching as input to then compute the full structural transformation cost, including edges and topology.
05 / 06 Machine Learning Integration

The penalty weights
are learnable.

GED's power lies in its cost function: each edit operation carries a penalty. Swapping an aromatic node for another aromatic should cost less than swapping it for a hydrophobic — because the first is biologically similar, the second is not.

Following García & Hernández, we treat these penalties as parameters optimized by machine learning — trained on labeled active/inactive molecule pairs — so the distance function reflects true biological relevance at the NS1 binding site, not arbitrary assumptions.
